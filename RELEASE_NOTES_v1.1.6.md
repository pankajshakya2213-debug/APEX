# APEX v1.1.6

## What's New
- Creator identity is now locked to Pankaj across chat, memory, and Settings personality text.
- APEX now treats the Settings user name as the current user's name instead of assuming the user is Pankaj.
- APEX avoids calling the user "boss" unless the saved user name or user preference says so.
- Chat replies can now use light, context-aware emojis when the conversation mood fits.

## Fixed
- Personality Matrix can no longer overwrite creator identity.
- Empty user name now falls back to a neutral "User" identity.
